## 坑一：scanf缓冲区问题（C语言）

```C
#include <stdio.h>

int main(int argc, char const *argv[])
{
    char c;
    do
    {
        printf("请输入");
        scanf("%c",&c);
    } while (c == 'y');
    return 0;
}


输出

请输入y
请输入
```

```C
#include <stdio.h>

int main(int argc, char const *argv[])
{
    char c;
    do
    {
        printf("请输入");
        rewind(stdin);
        scanf("%c",&c);
    } while (c == 'y');
    return 0;
}


输出
D:\sourcecode\C_SourceCode\test5>cd /d "d:\sourcecode\C_SourceCode\test5\" && gcc -fexec-charset=GBK -std=c99 test.c -o test && "d:\sourcecode\C_SourceCode\test5\"test
请输入y

请输入y
请输入y

```

每次循环scanf，都会在键盘缓冲区来读取东西

因为\n 不是%d，是%c类型的



## 坑二：const问题（C语言）

判断哪个被const了的标志是const在*的前面还是后面

```C
在*后表示指针所指向的值不可以被修改，指针地址可以被修改
const int *p1 = &i;
*p1 = 30; //ERROR (*p1)是const，不可以通过p1来修改p1所指向的值
i=30; //OK 可以直接修改i
p1 =&j //OK 可以将别的地址赋给p1
    
int const *p2 = &i;
*p2 = 30; //ERROR (*p2)是const，不可以通过p2来修改p2所指向的值
i=30; //OK 可以直接修改i
p2 =&j //OK 可以将别的地址赋给p2


```

```C
在*前表示指针地址不可以被修改，指针所指向的值可以被修改
int *const p3= &i;
*p3 = 10; //OK
p3++ //ERROR

```

```C
例：
typedef struct {
  int *array;
  int size;
} Array;

int *array_at(const Array *a, int index){
    a->array[index]= 10; //OK 因为array也是个指针所以并没有修改指针地址，而是修改了指针所指向的值
    a->size =10; //ERROR 因为在星号前面所以指针所指向的值不可以被修改
    return &(a->array[index]);
}

int *array_at(Array *const a, int index){
    a = (int *)malloc(sizeof(int) * 100);  //ERROR 在*后面不能修改a的地址
    a->array[index]= 10; //OK array本身是指针所以无限制，没有改变地址，改变了所指向的值
    a->size =10; //OK 可以修改a所指向的值
    return &(a->array[index]);
}
```



## 坑三：Mybatis判断问题（java语言）

在mybatis中 数值0为假，字符'0'为假，空字符''为假，其余为真

```java
int i = 0;
int j = 1;

i == '' ; 	//true
j == ''; 	//false

i == null; 	//false
j == null;	//false

i > -1;	 //true
j > -1;	//true


```
