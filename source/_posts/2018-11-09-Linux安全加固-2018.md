### 建议删除无用账户：
```
    lp,sync,shutdown,halt,news,uucp,operator,games,gopher
```
### 修改ssh端口

```
    vi /etc/ssh/sshd_config
            Port 2211
    vi /etc/sysconfig/iptables
            -A INPUT -m state --state NEW -m tcp -p tcp --dport 2211 -j ACCEPT
    service sshd restart
```

### 禁止root远程登录

```
    vi /etc/ssh/sshd_config
            PermitRootLogin no
    service sshd restart
```

### 允许管理员登录

```
    vi /etc/securetty
        CONSOLE = /dev/tty
```

### 设置空口令账户密码（$1账号$2密码$3为UID）

```
    awk -F: '($2 == ''){print $1}' /etc/passwd
    #根据结果利用passwd修改密码
```

### 清除root之外UID为0的用户（$1账号$2密码$3为UID）

```
    awk -F: '($3 == 0){print $1}' /etc/passwd
    #根据结果修改或者删除
```

### 缺省密码长度限制(至少八位)

```
    vi /etc/login.defs
            PASS_MIN_LEN 5 改为 PASS_MIN_LEN 8
```

### 限制能够su的用户

```
    vi /etc/pam.d/su
            auth        required    pam_wheel.so use_uid # 去掉‘#’号
    vi /etc/login.defs
            SU_WHELL_ONLY yes
```

### iptables 用法


```
    iptables -L -n     #查看iptables规则，如果查看不了执行su root
            iptables -A INPUT -p tcp --dprot 22 -j DROP
            iptables -A OUTPUT -p tcp --dprot 22 -j DROP    ##以上两条关闭22端口
            iptables -D INPUT 1
            iptables -D OUTPUT 1    #以上两条删除关闭22端口规则
            iptables -A INPUT -p icmp -j LOG    #记录icmp日志
            tail -f /var/log/syslog    #循环查看syslog
            iptables -I INPUT -p icmp -j REJECT  ##拒绝ping
            iptables -D INPUT -p icmp -j REJECT  ##删除规则
```

### 如果忘记了MySQL root密码，可以用以下方法重新设置：

```
    1. 停止mysqld； 
        /etc/init.d/mysql stop 
    2. 用以下命令启动MySQL，以不检查权限的方式启动； 
        safe_mysqld --skip-grant-tables &
    3.然后用空密码方式使用root用户登录 MySQL；
        MySQL -u root 
    4.修改root用户的密码；
        MySQL> update MySQL.user set password=PASSWORD('新密码') where User='root';  
        MySQL> flush privileges;  
        MySQL> quit 
    重新启动MySQL，就可以使用新密码登录了。
```
